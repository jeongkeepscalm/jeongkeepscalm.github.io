## 1강. 컴퓨터의 이해  
  
### 데이터와 정보 
  
* 데이터 : 관찰이나 측정을 통해 얻는 수치.  
  불을 기준으로 했을 때,  
  질적 데이터 : 붉다, 밝다, 뜨겁다...  
  양적 데이터 : 3500도  
* 정보 : 데이터를 처리하여 문제 해결에 도움이 되는 결과물.  
<img src="/assets/img/informationProcess.jpg" width="600px" />  

* 관찰 : 사물 또는 현상을 파악하는 행위(뜨겁다)  
* 측정 : 절대적인 기준에 맞춰 현상을 수치로 나타내는 과정.( 3500도(연속적 데이터), A급 화염(범주적 데이터) )  
  
### 컴퓨터의 개념  
  
* 컴퓨터 : 정의된 방법에 따라 입력된 데이터를 자동으로 처리하여 정보를 생산하는 기계.  
<img src="/assets/img/transistor.jpg" width="600px" />  

* 아날로그 : 사물이나 개념을 연속적인 물리량 값으로 표현(질적 데이터/양적 데이터)  
* 디지털 : 사물이나 개념을 이산적인 값으로 근사하여 표현(양적 데이터)  
* 알고리즘 
  문제를 풀기 위한 단계별 절차를 체계적 명령의 형태로 기술한 것이다.  
  주어진 명령어를 처리하는 컴퓨터에게 문제를 해겨하도록 만드는 정형화된 절차.  
<img src="/assets/img/programingLanguage.png" width="600px" /> 
  
* 컴파일러/인터프리터 : 고급 언어로 작성된 프로그램은 기계어 밖에 인식할 수 없는 컴퓨터에 의해 실행이 불가능하다. 그래서 컴파일러/인터프리터가 기계어로 변환해 컴퓨터에게 전달한다.

## 2강. 컴퓨터의 구성
  
### 하드웨어  
  
폰 노이만 구조(사람의 정보처리 과정을 컴퓨터에도 동일하게 적용)  
<img src="/assets/img/hardware.jpg" width="600px" /> 
  
* 입력장치 : 명령과 데이터를 컴퓨터가 처리할 수 있는 2진수 형태로 변환시켜 전달하는 장치.
* 출력장치 : 정보 처리 결과를 인간이 인식 가능한 형태의 데이터로 보여주는 장치.
    특수출력장치  
      HUD(Head-Up Display) : 현실 정보 + 가상 정보. AR(Argumented Reality)  
      HMD(Head-Mounted Display) : 가상 정보. VR(Virtual Reality)  
* 기억장치 : 명령과 데이터를 기억(저장)하는 하드웨어. 역할에 따라 주기억장치와 보조기억장치로 나뉜다. 자주 사용하지 않는 데이터는 보조기억장치에 두고, 현재 사용하고 빨리 처리해야하는 데이터는 주기억장치에 
  <img src="/assets/img/romram.jpg" width="600px" />   
  
  * 주기억장치  
    ROM(Read Only Memory) : 한 번 데이터를 저장하면 불변하기에, 컴퓨터 부팅 시 필요한 데이터,정보(바이오스)를 저장한다.  
    RAM(Randon Access Memory) : 데이터를 임시적으로 저장하는 공간.  
  * 보조기억장치  
    자기디스크(하드디스크)  
    광학디스크  
    플래시 드라이브  
    플로피디스크  
  * 특수 기억(저장)장치 / SSD(Solid Satate Drive) : RAM을 이용한 저장장치이다. 속도가 빠르고, 외부 충격에 강하고 전력소모가 적다. 하드디스크를 SSD로 변경하면 컴퓨터가 빨라질 수 있다.  
  
* 제어와 연산장치 : 명령어와 데이터를 읽고 데이터를 연산/처리 하는 장치.   
  CPU 
    레지스터(Register) : CPU 내부에서 초고속으로 데이터를 저장하거나 얽어낸다.  
    제어 유닛(Control Unit)  
    연산 논리 유닛(ALU)  
    <img src="/assets/img/cpu.jpg" width="600px" />   
  
### 소프트웨어  
  
소프트웨어 : 각각의 고유한 기능을 수행하는 하드웨어가 언제 어떻게 동작하여 문제를 해결할 수 있는지 지시하는 명령어 집합. 알고리즘을 프로그래밍 언어로 구현한 결과물.  
  * 응용 소프트웨어(애플리케이션) : 사용자의 업무나 목적에 맞게 문제 해결을 위한 처리 절차를 표현한 명령어 집합  
  * 시스템 소프트웨어 : 하드웨어를 제어,관리하여 응용 소픝웨어를 실행할 수 있는 환경을 제공한다.  
    1. 운영체제 : 사용자가 컴퓨터를 효율적으로 운영/관리/사용할 수 있도록 하드웨어를 제어하는 소프트웨어. 응용 소프트웨어가 효과적으로 작동할 수 있는 환경을 조성한다.   
      사용자인터페이스 : 컴퓨터와 사용자를 매개하여 상호작용하는 방법을 제공한다.(GUI, CLI)  
      프로세스관리  
      네트워크 인터페이스 : 컴퓨터는 네트워크를 통해 상호 데이터 교환한다.  
      기억/저장장치 관리
    2. 컴파일러 : 소스코드를 컴퓨터가 이해할 수 있는 기계어로 번역하는 소프트웨어.  
    3. 유틸리티 : 부가적인 기능을 제공하여 사용자가 컴퓨터를 효율적이고 편리하게 관리할 수 있도록 지원해주는 소프트웨어.  
  
* QUIZ  
  
1. 플로피디스크, 자기디스크, 광학디스크는 전기를 사용하지않고 데이터를 저장하는데, 
구체적으로 어떻게 데이터를 저장하는지 알아보자.  
  플로피디스크 : 디스크 표면에 있는 자성 재료에 미세한 자기 변화를 일으켜 저장한다. 전기 신호를 자기 신호로 변환하여 데이터를 저장. 자기 신호를 전기 신호로 변환하여 데이터 읽는다.   
  자기디스크 : 자기디스크는 금속 디스크를 사용하고 고속 회전하며 자기 신호를 읽고 쓸 수 있다.   
  광학디스크 : 레이저 기술을 사용하여 데이터를 저장한다. (CD, DVD, Blu-ray)  
  
2. GUI / CLI 기반의 운영체제는 무었이 있는지 찾아보자.  
GUI : Windows, macOS ...  
CLI : DOS, Linux, UNIX ...

## 3강. 파이썬의 이해  
  
파이썬은 쉘 스크립트 언어용으로 개발된다.  
쉘(Shell) : 사용자가 입력한 명령어를 해석해서 운영체제 내부의 커널에 전달하는 명령 해석기 창.  
스크립트 : 운영체제가 어떤 순서로 동작해야 하는지 명령어를 집합으로 만들어 놓은 형태. 
DARPA : 미 국방성 연구기관으로 이곳에서 최초의 인터넷이 탄생했다.   
  
파이썬의 탄생  
  1. 다중 프로그래밍 패러다임 채용(명령형, 절차적 프로그래밍)  
    패러다임 : 현실 세계를 어떠한 방식으로 모델링하는 것에 대한 여러 관점들을 규칙화하여 만든 것.  
  2. 다목적 활용  
    응용프로그램과 웹, 백 엔드 개발, 사물 인터넷 분야 뿐만 아니라 교육적인 목적으로도 활용.  
  
<img src="/assets/img/pythonUpdate.jpg" width="600px" />   
  
### Pythonic (파이썬의 장점)  
  1. 독립적 : 운영체제와 상관없이 실행가능.  
  2. 오픈소스  
  3. 인간적 : 자연어와 유사.  
  4. 신속성 : 여러 라이브러리와 프레임워크가 만들어져있어 곧 바로 프로그래밍할 수 있다.   
  5. 직관적
  
<img src="/assets/img/pythonLibrary.jpg" width="600px" />   
NumPy, Numba, Sympy : 숫자/수치 해석을 위한 기능을 내장한 라이브러리.  
Ipython, Jupyter notebook, Jupyter lab : 개발 환경 제공.  
Flask, dj : 특정 순서대로 개발하기 위한 프레임워크 제공.  
Matplotlib, Bokeh, pandas : 수치와 관련된 함수를 만들고 그래프화 할 수 있는 라이브러리.    
astropy, biopython, sunpy : 각종 과학기술분야에서 특화되서 사용할 수 있는 기능들을 갖추고 있는 라이브러리.  
PyTorch, TensorFlow : 딥러닝을 위한 라이브러리. 
  
### 파이썬 단점  
  1. c 나 자바 등으로 작성된 프로그램보다 느린 속도.  
  2. 완전한 애플리케이션 단독 개발이 불가능하다.  
    파이썬은 쉘 스크립트 언어 용으로 개발된다.  
    모바일 앱 등 응용 애플리캐이션 개발 불가능하다.(Rust 또는 Go라는 프로그래밍언어로 개발이 가능하다.)  
  
### 파이썬 실행 환경  
  1. 플랫폼에 독립적 : 윈도우, 리눅스, 유닉스, 맥OS 등 다양한 운영체제(플랫폼)에서 별도의 컴파일 없이 실행 가능하다.  
  2. 인터프리터식 객체지향적 : CPython, PyPy, Cython, Jython 등 다양한 인터프리터 환경 사용가능하다. 프로그램을 객체로 모델링한다.  
  3. 동적 타이핑 : 타입을 지정해 줄 필요없이 알아서 타입을 관리해준다.  
  4. 대화형 언어 : 작성한 코드에 대한 수행 결과를 바로 확인하고 디버깅하면서 코드 작성 가능하다.  
  
### CPyton : C 언어로 개발된 파이썬 인터프리터  
  컴파일러의 유형  
  1. 셀프 호스팅 컴파일러 : 부트스트래핑 단계를 통해 자신의 언어로 작성한 컴파일러이다.  
  부트스트래핑 사전적 의미 : 신발끈을 묶는다.  
  ex ) C나 자바는 자신의 언어로 컴파일러/인터프리터를 만든다.  
  2. 소스 대 소스 컴파일러 : 타 언어로 작성한 컴파일러.  
  => CPython은 소스 대 소스 컴파일러로 만들어진 인터프리터를 사용한다.

### 파이썬 프로그래밍 환경  
  
* IDLE(Integrated Development Environment, 통합개발환경)  
  파이썬 인터프리터를 설치하면 기본으로 포함된 통합개발환경이다.  
  파이썬과 Windows환경에서 앱을 만들 수 있는 Tkinter GUI 툴킷으로 개발환경.  
  
* Jupyter NoteBook  
  오픈소스 기반의 웹 플랫폼  
  인터프리터로 IPython을 사용하여 전통적인 소스코드 -> 컴파일 -> 실행 방식에서 벗어나 웹 기반 대화형 개발 및 실행 환경 제공.  
  문서화하여 공유가 편리하다.  
  마크다운을 이용하여 코드 관련 타이틀, 설명 등 작성 가능하다.  

* Colab  
  클라우드 기반 주피터 노트북 개발 환경을 제공.    
  주피터 노트북 + 구글 드라이브를 결합한 서비스.  
  데이터 분석 및 딥러닝 연산 등 고성능 컴퓨팅 리소스 활용 가능.  
  **구글의 클라우드 서비스인 GCP**로부터 자원을 할당받아 코드를 실행하고 결과를 리턴한다.
