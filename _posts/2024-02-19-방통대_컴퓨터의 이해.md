## 1강. 정보사회와 컴퓨터  
  
#### 키워드  
* 유비쿼터스 : 컴퓨터와 네트워크가 산재되어 있는 환경.  
* 개인용 컴퓨터 : 1980년대 초 IBM-PC와 운영체제인 MS-DOS 보급으로 급속히 발전(데스크톱PC, 노트북PC, 넷북, PDA 태블릿 PC, 슬레이트PC 등.. )  
* 메인프레임  
  1. 다중 사용자 환경을 지향하도록 설계.  
  2. 수많은 연산을 동시해 수행하는 다중의 프로세서들을 갖추고 있다.  
  3. 데용량의 데이터 처리나 네트워크 관리를 위한 서버로 많이 사용.  
* 데이터 : 정보로 추출되어지기 전의 가공되지 않은 물질.  
* 정보 : 데이터가 어떤 매개체에 의해 집대성되고 의미있는 것으로 가공 처리 된 것.  
* 멀티미디어 : 컴퓨터를 기반으로 문자, 오디오, 비디오, 그림, 애니메이션 등을 종합적으로 입출력하며 처리하고 저장할 수 있는 매체. 인터넷 기반으로 양방향 통신과 주문형(on-demand) 및 대화형(interactive) 매체가 보편화 되었음.  
* 컴퓨터시스템의 세 가지 주요 구성장치
  * 입력장치  
  * 처리장치  
    1. 중앙처리장치  
      - 제어장치  
      - 연산장치  
    2. 주기억장치  
  * 출력장치  
* 프로그램 : 주기억장치에 저장된 일련의 명령어들로 컴퓨터시스템에서 일어나야 할 동작들을 규정한 것이다.  
  
#### 정리
* 컴퓨터, 반도체기술, 통신기술, 인터넷, 멀티미디어, 광통신 네트워크에 의하여 산업과 사회의 각 분야를 크게 변화시키는 정보화 혁명을 가져왔다. 정보화사회(Information Society) 라는 용어 탄생.  
* 초고속 전송이 가능한 광섬유 네트워크기술과 무선랜(wireless LAN) 기술 등에 의한 유비쿼터스(ubiquitous) 시대는 스마트폰, 웨어러블 기기, 사물인터넷 등으로 더욱 활성화되었다.  
* 컴퓨터시스템의 하드웨어는 물리적인 요소로 일종의 기계적인 장치를 말하며, 소프트웨어는 논리적인 요소로 일종의 프로그램의 집합을 말한다.  
* 소프트웨어는 사람이 컴퓨터에게 작업을 지시하거나 하드웨어와 이용자 사이에서 이용자가 하드웨어를 의식하지 않고 쉽게 이용할 수 있도록 매개체 역할을 수행하는 일종의 프로그램들의 집합이다.  
* 컴퓨터시스템의 세 가지 기본기능은 데이터의 가감승제를 위한 산술연산, 두 수의 크기를 비교하는 논리연산, 그리고 데이터를 받아들이거나 결과를 출력하는 입출력 동작이다.  
* 컴퓨터는 크기, 성능, 가격, 사용자 수를 기준으로 개인용 컴퓨터, 서버시스템, 메인프레임, 전용컴퓨터, 슈퍼 컴퓨터 등으로 분류된다.  
* 운영체제는 데스크톱 PC나 노트북 PC, 슬레이트 PC는 MS-DOS와 Windows 시리즈를 거쳐 2015년 Windows 10, 2021년 Windows 11이 출시되었다. 또한 PDA 운영체제로는 팜(Palm) OS, Windows CE 등이 있으며, 스마트폰용 운영체제로는 대표적으로 iOS, 안드로이드 등이 있다.  
* 1990년대 초반 이후 펜티엄 프로세서나 알파 프로세서 등과 같은 고성능의 마이크로프로세서를 장착한 다양한 종류의 서버시스템들이 출시되어 많은 영역에서 메인프레임이나 슈퍼컴퓨터의 기능을 대신하고 있다.

## 2강. 컴퓨터와 통신산업의 발전
  
#### 키워드  
* ABC : 최초의 전자식 디지털 컴퓨터 from 아타나소프, 베리
* ENIAC : 대형 전자식 디지털 컴퓨터 with 진공관 from 모클리, 에커트  
* UNIVAC I : 최초의 상업용 컴퓨터 (미국 조사통계를 위해 사용. 데이터 처리용)  
* TRADIC : 첫 번째 트랜지스터를 사용한 컴퓨터 (1954 in 벨 연구소)
* 직접회로 : 실리콘 칩 위에 많은 트렌지스터들과 전자회르들을 결합한 것. from 킬바이, 노이스
* 캐시메모리 : 컴퓨터 속에 장착해 속도를 빠르게하는 임시메모리. (램캐시, 내부캐시, 디스크캐시 등.. )  
* 유후 상태 : CPU 나 하드웨어 장치에서 다음에 수행할 명령어와 작업을 기다리는 동안 작업을 수행하지 않고 대기하는 상태.  
* 배비지의 해석기계 : 우리가 사용하고 있는 컴퓨터 시스템의 5대 기능과 유사(1834에 설계)  

#### 정리 
* 1세대 진공관 -> 2세대 트랜지스터(TRADIC, 벨 연구소) -> 3세대 직접회로 -> 4세대 고밀도 직접회로  
* 2세대에는 고급프로그래밍 언어를 사용한 프로그래밍 기법과 실시간 시스템, 운영체제를 사용했다.  
* 3세대에는 마이크로 프로그래밍, 다중프로그래밍, 병렬처리, 운영체제 등이 크게 발전했다.  
* 4세대에는 1970년대 초 이후를 말하며, 고밀도 및 초고밀도 직접회로를 사용한 시기이다.  
* 1990년대  
  1. 마이크로프로세서와 네트워크 기술의 발달 -> 분산계산, 병렬계산 보편화  
  2. 인터넷이라는 통신망이 전 세계적으로 사용되기 시작.  
  3. 개인 컴퓨터의 성능이 크게 발전.  
* 컴퓨터시스템을 이용한 자료처리  
  1. 일괄처리 : 자료를 모아서 한꺼번에 처리.  
  2. 거래지향처리 : 자료가 발생하자마자 바로 처리.  
* 메모리 직접도는 광소자, 조셉슨 소자, 갈륨비소 소자 등으로 더욱 향상될 것이며, 광디스크나 램 디스크, 모바일 D램 등을 이용한 보조기억장치도 소형화, 경량화 되면서 저장용량은 크게 증가할 것이다.  
* 5세대 컴퓨터는 인공지능형(AI : Artificial Intelligence) 컴퓨터로 전기대신 빛으로, 반도체 대신 초전도체로 움직일 것이며, 인간의 신경회로망을 닮은 신경망 컴퓨터로 발전될 것이다.

## 3강. 처리장치와 데이터처리  
  
#### 키워드  
* 처리장치  
  - 중앙처리장치(CPU) : 프로그램 명령어를 해석하고 계산하고 데이터를 이동시키며, 시스템 입출력조작을 제어한다. 
    - 산술논리연산장치(ALU)  
    - 제어장치  
  - 주기억장치 : 입력된 데이터를 기억시키고 바이트, 워드 단위로 기억장치를 분할해서 주소를 할당하여 사용한다.  
* 비트(bit : binary digit) : 주기억장치 내부에 기억된 데이터의 최소 기본단위.  
* ASCII : 8비트로 구성(1비트는 검증비트(parity bit)). 128개 문자까지 표현가능.  
* UNICODE : 세계 각국의 언어를 좀 더 쉽게 표현할수 있도록 개발된 16비트 코드체계.  
* 레지스터 : CPU 안에 들어있는 소규모 데이터 기억장치. 내부 버스가 연산장치와 레지스터를 연결해준다. 데이터 저장 속도는 주기억장치에 비해 훨씬 빠르다.   
  - 명령 레지스터  
  - 번지 레지스터  
  - 인덱스 레지스터  
  - 누산기  
* 버스 : 정보를 전송하는 통로(통신 시스템)  
* RAM(random access memory / read-and-write memory) - 읽고 기록하는 메모리.  
  - SRAM(Static RAM) : 전원이 공급되는 한 내용이 그대로 유지된다. 캐시메모리로 이용된다.  
  - DRAM(Dynamic RAM) : 내용 소멸을 방지하기 위해 계속적으로 리프레싱(refreshing)이 요구된다. 주기억장치로 많이 이용된다.  
* ROM(read only memory) - 읽기만 가능하다. 메모리를 만드는 과정에서 데이터를 저장한다. 컴퓨터 부팅에 관한 것들이 rom에 들어있다.  
* PROM : 생산될 때, 비어있으나 사용자가 한 번에 한해 내용을 기록할 수 있는 메모리.   
* 플래시메모리 : 최종 사용자가 쉽게 내용을 변경할 수 있다. 비파괴메모리 기술을 적용한 메모리의 한 형태. 새로운 입출력장치와 저장장치를 가진다. 전력이 없어도 기억이 된다(usb)  
  
#### 정리  
* 컴퓨터 프로그램의 제어를 받아 처리하는 컴퓨터시스템은 주기억장치에 기억된 데이터를 처리하고 출력한다.  
* 수치정보는 2진수로 표시된다. 값의 크기를 표시할 수 있는 비트수가 일정할 때, 고정장 워드머신이라고 한다.  
* 주기억장치에서 사용되는 기억용량의 단위는 Byte, KB, MB, GB, TB, PB이고 기억장치의 호출시간을 측정하는 단위는 msec, μsec, nsec, psec이다.  
* 주기억장치의 각 바이트는 그것에 관련된 고유한 주소를 갖는데, 주기억장치에 있는 이 주소는 각 바이트를 식별한다.  
* 데이터가 주기억장치에 기록되면, 그 데이터는 참조할 수 있고 조작할 수 있다.  
* 기본적인 처리과정  
  1. 입력 : 데이터를 주기억장치에 기억.  
  2. 처리 : 주기억장치에 있는 데이터를 처리.  
  3. 출력 : 처리된 결과를 보조기억장치로 출력하거나 우리가 볼 수 있는 형태로 출력.  
* 필드(컬럼) < 레코드(한 행) < 파일(여러 행)  
* 출력  
  - 프린터나 모니터를 이용한 출력  
  - 보조기억장치(자기테이프나 자기디스크 등..)를 이용한 출력. 다음에 다른 프로그램에서 입력으로 읽어들여 계속 사용할 수 있도록 하기 위한 것이다.

## 4강. 컴퓨터의 입출력  
  
#### 키워드  
* thin client : 대부분의 계산 및 처리, 저장기능은 서버가 담당. 사용자에게 GUI만 제공하는 필수 요소만으로 구성된 저사양 컴퓨터.  
* fat client : 각종 응용브로그램을 내장. 서버와 독립적인 처리 능력을 갖춤.  
* 광학마크인식(OMR) : 정해진 위치에 표시한 마크를 인식하여 데이터를 입력하는 기술  
* 광학문자인식(OCR) : 인쇄되거나 손으로 쓴 글자를 인식하여 입력하는 기술  
* 자기잉크문자인식(MICR) : 자기 잉크로 인쇄된 글자를 인식하여 입력하는 기술  
* 매트릭스 코드 : 2차원 영역에 패턴을 나열하여 정보를 표현하는 방식으로, 2차원 바코드라고 부르기도 함. 바코드에 비해 더 많은 정보를 표현할 수 있음  
* RFID : 전파를 이용하여 정보를 전달하는 기술  
* 체크 디지트 : 데이터 입력 과정에서 발생되는 오류를 검사하기 위해 추가된 숫자  
* 헤드 마운티드 디스플레이(HMD) : 작은 광학 디스플레이 장치를 통해 영상을 두 눈에 전달할 수 있도록 머리에 쓰는 디스플레이 장치  
  
#### 정리  
* 기계가 읽을 수 있는 원시문서  
  - OMR  
  - OCR  
  - MICR  
  - 바코드  
* 데이터 입력 시 하위 편집과정을 통해 올바른 데이터가 입력되로록 해야 한다.  
  - 자료형  
  - 값의 타당성  
  - 갑의 범위의 적정성  
  - 일관성  
* 디스플레이 모니터 : 대화식 처리에서 가장 많이 사용되는 대표적인 출력장치  
  - LCD  
  - PDP  
  - OLED

## *************** 운영체제 정리 ***************
  
* 커널모드(슈퍼바이저 모드) : 하드웨어를 직접 제어할 수 있는 CPU 명령어를 사용할 수 있는 모드. 운영체제의 커널이 동작하는 모드.  
* 사용자모드(보호 모드) : CPU 명령어를 사용할 수 없는 모드. 응용프로그램이 동작하는 모드.  
  
* 커널(kernel) : 커널 모드에서 동작하는 운영체제의 핵심 요소. 응용프로그램과 하드웨어 수준의 처리사이의 가교역할. '커널이 운영체제다' 라고 생각할 수 있지만 구성 방식에 따라 차이가 있다.   
* 일체형커널 ( 커널 == 운영체제 )  
	운영체제의 모든 서비스가 커널 내 포함.   
	커널 내부 요소들의 효율적으로 상호작용.  
	한 요소라도 오류가 발생하면 시스템 전체에서 장애 발생 가능.  
	( UNIX, Linux.. )  
* 마이크로커널   
	운영체제 요소의 대부분을 커널 외부로 분리.  
	커널 안에는 가장 핵심적인 요소들만 있다. ( 메모리 관리, 멀티태스킹, 프로세스 간 통신(IPC) 등.. )  
	새로운 서비스를 추가하여 운영체제를 확장하기 쉽다.  
	유지보수가 용이하여 안정성이 우수하다.   
	커널 외부 요소들 사이는 IPC가 필요하여 성능저하가 발생한다. (외부에서 내부의 IPC 를 통해서 가야하므로)  
  
* 시스템 호출   
	응용프로그램이 하드웨어에 대한 제어가 필요할 때, 이용한다.  
	운영체제에 서비스를 요청하는 메커니즘.  

#### 운영체제 구성
  
* 프로세스 관리자 ( 실행 중인 프로그램을 관리한다. )  
	프로세스의 생성 및 삭제  
	CPU 할당을 위한 스케줄 결정  
	프로세스의 상태를 관리하며 상태 전이 처리  
	메모리에 올라와있는 프로그램의 순서를 스케줄링이 결정한다.   
* 메모리 관리자  
	차지하고 있는 메모리와 겹치지 않게 메모리 공간을 부여해서 할당 및 회수한다.  
	메모리 공간을 보호한다.   
* 장치 관리자   
	컴퓨터 시스템의 모든 장치에 할당, 작동, 반환해준다. ( 하드 디스크, SSD, 키보드, 마우스, 프린터 등.. )  
* 파일 관리자  
	저장장치의 공간 관리  
	파일의 접근 제한 관리  
  
#### 운영체제의 유형
  
* 일괄처리 운영체제 (batch processing)  
	작업을 모아서 순서대로 처리하는 방식  
	전체적인 작업 처리 속도가 향상됨  
	나중에 들어온 중요도 높은 작업은 우선순위에 밀려 기다리게 된다.  
* 시분할 운영체제 (Time-sharing) == 대화형 운영체제 (interactive)  
	각 프로그램을 한 번에 조금씩 수행하는 방식  
	응답시간이 일괄처리 운영체제보다 크게 단축된다.  
* 실시간 운영체제 (real-time) (RTOS)
  원하는 시간 내에 프로그램 결과를 얻을 수 있는 방식. 시분할 운영체제는 응답시간이 빨라졌지만 실시간 운영체제는 최종적인 결과물을 받는 시간이 빨라졌다.  
  처리결과가 현재의 결정에 영향을 주는 환경에서 사용된다. ( 미사일 제어 시스템, 증권거래 관리 시스템 등.. )  
  우선순위가 높은 작업을 최우선처리할 수 있는 기법.
* 분산 운영체제  
  분산 시스템을 관리하기 위한 운영체제.
  분산 시스템 : 2개 이상의 컴퓨터 시스템이 네트워크로 서로 연결되어 서로의 자원을 이용하는 시스템  
  다른 컴퓨터 시스템의 자원을 이용하는 것이 마치 자신 컴퓨터 시스템에 있는 자원을 이용하는 것처럼 가능하게 한다. 
  
## 2강. 프로세스와 쓰레드  
  


#### 프로세스 개요    
  
  Hwp.exe 처럼 저장장치에 있을 때는, 프로그램에 관련된 명령어 목록이 들어있다고 보면 된다. (용량이 매우 작음)  
  Hwp.exe 가 실제 프로세스가 되면, 그 프로그램 자체에 부가적인 다양한 데이터들이 더 필요해서 메모리 용량이 훨씬 커지게 된다.  
  
* 프로세스
  실행중인 프로그램  
  동작을 하는 능동적 개체  
  운영체제로부터 자원을 할당 받아 동작한다.  
* 프로그램 : 동작하지 않는 정적, 수동적 개체  
=> 정적이였던 프로그램이 메모리 상에 올라와서 여러가지 자원을 할당 받고, 최종적으로 CPU도 할당받아서 실제 명령을 실행하는 동작을 한다.  
  
* 운영체제(프로세스 관리자)가 처리하는 작업
  프로세스를 생성 및 종료  
  프로세스를 실행시키기 위한 스케줄링 작업  
  프로세스의 상태 관리  
  
* 메모리 구조  
  코드 영역  
    프로그램 자체  
  데이터 영역  
    프로그램 실행 시 필요한 데이터를 보관한다.( 상수나 변수의 값, 서브프로그램 호출 상태 등.. )  
      정적 데이터 영역  
      스택 영역  
      힙 영역 
  
* 프로세스 제어 블록(PCB : Process Control Block) : 운영체제가 프로세스를 관리하기 필요한 정보를 가지고 있다.  
  프로세스번호(PID)  
  프로세스 상태  
  프로그램 카운터(PC) : 다음에 어떤 처리를 해야 하는지 알 수 있다. 제어의 흐름을 볼 수 있다.    
  레지스터 : 여러 프로세스가 번갈아 실행되는 경우 레지스터에 저장되어있던 값을 활용한다.  
  메모리 관리 정보  
  프로세스 우선순위  
  회계 정보 ...  
  
* 프로세스 상태  
  <img src="/assets/img/processStatus.jpg" width="800px" />
* 디스패치 : 특정 프로세스에 cpu를 할당해주는 과정  
  
* 프로세스 생성 방법  
  1. 사용자가 직접 프로그램 실행  
  2. 한 프로세스가 다른 프로세스 생성(부모프로세스가 자식프로세스 생성) : 프로세스 생성 시스템 호출 이용  
  
  * UNIX, Linux 일 경우  
  UNIX, Linux는 fork() 라는 시스템호출을 사용한다.  
    fork() : 부모프로세스를 복제해서 PID만 다른 자식프로세스를 생성한다. 자식프로세스의 PID를 return 한다.  
    fork()된 자식프로세스에 다른 일을 시키고 싶으면, fork() 다음에 exec()라는 시스템호출 하면 된다.  

  ```java
  // 메모리구조 내 코드영역
  // 만약 자식프로세스라면 program2라는 프로그램을 실행시켜라. 
  pid = fork();
  if (pid == 0) 
    exec(program2);
  ```

  * Windows 일 경우  
    CreateProcess()  
      자식프로세스는 처음부터 새로운 프로그램으로 생성
    
  ```java
  // 메모리구조 내 코드영역
  // 해당 자식프로세스는 곧바로 program2를 실행시켜라.
  CreateProcess(program2, ... ,&pid);
  ```
  
* 프로세스 종료 방법  
  프로세스가 모든 처리를 완료할 때( 정상적 종료 )  
  부모프로세스가 자식프로세스를 강제 종료할 수 있다.    
    자식의 PID를 이용해서 프로세스 종료 시스템 호출한다.  
  부모 프로세스가 종료되는 경우 운영체제가 자식 프로세스를 종료할 수 있다.

#### 쓰레드  
  
* 쓰레드(Thread)
  프로세스 내에 다중처리를 위해 제안된 개념  
  디스패칭의 단위  
  하나의 프로세스에는 하나 이상의 쓰레드가 존재한다.  
  전통적인 프로세스와는 다르게 쓰레드가 PC를 가지고 있다.  
  실행에 필요한 최소한의 정보만 가진다.  
    PC를 포함한 레지스터 값  
    상태정보  
    스택 영역  
  나머지 정보는 프로세스에 두고 다른 쓰레드와 공유

* 다중 쓰레드로 구성된 프로세스  
  멀티 CPU 또는 멀티코어 컴퓨터 시스템
    코어 3개 == CPU 3개. 즉, 3개의 쓰레드가 동시에 처리 가능.
